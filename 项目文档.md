# BBS论坛系统项目文档

## 项目概述

BBS论坛系统是一个基于Spring Boot + MyBatis的轻量级论坛应用，支持用户注册登录、发帖回复、收藏管理、积分系统等完整的论坛功能。系统采用前后端分离架构，前端使用Thymeleaf模板引擎，后端使用Spring Boot框架。

## 系统架构

### 技术栈

#### 后端技术
- **核心框架**: Spring Boot 2.7.18
- **持久层框架**: MyBatis 2.3.1
- **数据库**: MySQL 8.0.33
- **连接池**: Druid 1.2.20
- **模板引擎**: Thymeleaf
- **构建工具**: Maven
- **Java版本**: 1.8
- **云存储**: 阿里云OSS SDK 3.18.1
- **HTTP客户端**: OkHttp 4.12.0

#### 前端技术
- **模板引擎**: Thymeleaf
- **样式框架**: Bootstrap
- **图标库**: Font Awesome

## 数据库设计

### 用户表 (user)
```sql
CREATE TABLE user (
    uid INT PRIMARY KEY AUTO_INCREMENT,
    uname VARCHAR(50) UNIQUE NOT NULL,
    upwd VARCHAR(50) NOT NULL,
    ustate INT DEFAULT 1,
    ucreatetime DATETIME DEFAULT CURRENT_TIMESTAMP,
    level INT DEFAULT 1,
    phone VARCHAR(20),
    career VARCHAR(100),
    address VARCHAR(200),
    score INT DEFAULT 0
);
```

### 帖子表 (post)
```sql
CREATE TABLE post (
    pid INT PRIMARY KEY AUTO_INCREMENT,
    ptitle VARCHAR(200) NOT NULL,
    pbody TEXT NOT NULL,
    psendtime DATETIME DEFAULT CURRENT_TIMESTAMP,
    lastreplytime DATETIME DEFAULT CURRENT_TIMESTAMP,
    uid INT NOT NULL,
    category VARCHAR(20) NOT NULL,
    is_sticky INT DEFAULT 0,
    prize INT DEFAULT 0,
    path VARCHAR(500),
    FOREIGN KEY (uid) REFERENCES user(uid)
);
```

### 回复表 (reply)
```sql
CREATE TABLE reply (
    rid INT PRIMARY KEY AUTO_INCREMENT,
    pid INT NOT NULL,
    uid INT NOT NULL,
    replymessage TEXT NOT NULL,
    replytime DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_top INT DEFAULT 0,
    FOREIGN KEY (pid) REFERENCES post(pid),
    FOREIGN KEY (uid) REFERENCES user(uid)
);
```

### 收藏表 (favorite)
```sql
CREATE TABLE favorite (
    fid INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL,
    pid INT NOT NULL,
    favtime DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (uid) REFERENCES user(uid),
    FOREIGN KEY (pid) REFERENCES post(pid)
);
```

## 功能模块详解

### 1. 用户管理模块

#### 核心功能
- **用户注册**: 支持用户名、密码、联系信息注册
- **用户登录**: 基于Session的登录认证
- **个人信息管理**: 修改密码、更新个人资料
- **账户注销**: 永久删除用户账户
- **权限管理**: 管理员和普通用户分级

#### 技术实现
- 控制器: `UserController.java`
- 服务层: `UserService.java`, `UserServiceImpl.java`
- 数据访问: `UserMapper.java`
- 安全特性: 明文密码存储（教学演示用）

### 2. 帖子管理模块

#### 核心功能
- **发帖功能**: 支持多板块发帖，可选封面图片
- **帖子浏览**: 分页显示，支持置顶排序
- **内容管理**: 编辑帖子内容，删除帖子
- **板块分类**: 娱乐、技术、美食、旅游、问题
- **置顶功能**: 管理员可将重要帖子置顶

#### 技术实现
- 控制器: `PostController.java`
- 服务层: `PostService.java`, `PostServiceImpl.java`
- 数据访问: `PostMapper.java`
- 文件上传: 阿里云OSS集成

### 3. 回复管理模块

#### 核心功能
- **回复功能**: 对帖子进行回复
- **置顶回复**: 帖子创建者可置顶回复
- **积分奖励**: 置顶回复可获得积分
- **时间排序**: 置顶回复优先显示

#### 技术实现
- 控制器: `ReplyController.java`
- 服务层: `ReplyService.java`, `ReplyServiceImpl.java`
- 数据访问: `ReplyMapper.java`

### 4. 收藏管理模块

#### 核心功能
- **添加收藏**: 收藏感兴趣的帖子
- **取消收藏**: 移除已收藏的帖子
- **收藏列表**: 查看个人收藏的所有帖子
- **状态显示**: 实时显示帖子的收藏状态

#### 技术实现
- RESTful API设计
- 异步状态更新
- 数据库关联查询

### 5. 积分系统

#### 核心功能
- **积分展示**: 个人中心显示当前积分
- **奖励机制**: 问答板块发帖设置奖励积分
- **自动发放**: 置顶回复时自动发放积分
- **积分统计**: 用户积分累计和展示

#### 积分规则
- 问答板块发帖时可设置1-10分奖励
- 帖子创建者置顶回复后，回复者获得相应积分
- 积分只能由系统自动发放，不可手动修改

### 6. 管理员功能

#### 核心功能
- **用户管理**: 禁言/解禁用户，注销用户账户
- **帖子管理**: 删除不当帖子，置顶/取消置顶
- **系统监控**: 查看系统运行状态
- **数据管理**: 维护系统数据完整性

#### 权限控制
- 管理员级别: level = 0
- 普通用户: level = 1
- 分级权限验证

## API接口文档

### 用户相关接口

#### 用户注册
```
POST /register.do
参数: uname, upwd, phone, career, address
返回: JSON格式的成功/失败信息
```

#### 用户登录
```
POST /login.do
参数: uname, upwd
返回: 重定向到首页或登录页
```

#### 用户退出
```
GET /logout.do
返回: 重定向到登录页
```

#### 个人信息页面
```
GET /person.do
返回: 个人信息页面视图
```

### 帖子相关接口

#### 首页帖子列表
```
GET /
参数: page (可选，默认第1页)
返回: 首页视图，包含分页帖子列表
```

#### 我的帖子管理
```
GET /myPosts.do
返回: 我的帖子管理页面
```

#### 更新帖子内容
```
POST /updatePostContent.do
参数: pid, pbody
返回: JSON格式的更新结果
```

### 收藏相关接口

#### 添加收藏
```
GET /favorite/add/{pid}
返回: JSON格式的收藏结果
```

#### 取消收藏
```
GET /favorite/remove/{pid}
返回: JSON格式的取消收藏结果
```

#### 收藏列表
```
GET /favorite/list
返回: 收藏列表页面视图
```

### 管理员接口

#### 用户禁言
```
GET /ban/{uid}
返回: 重定向到用户管理页面
```

#### 用户解禁
```
GET /unban/{uid}
返回: 重定向到用户管理页面
```

#### 帖子置顶
```
GET /toggleSticky/{pid}/sticky
返回: JSON格式的置顶结果
```

#### 取消置顶
```
GET /toggleSticky/{pid}/unsticky
返回: JSON格式的取消置顶结果
```

## 开发规范

- 遵循Java编码规范
- 使用MyBatis注解和XML混合开发
